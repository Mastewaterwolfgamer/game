Guide To RegExp

What is Regex?

Regex (or RegExp) stands for Regular Expression and is a very powerful and versatile tool for searching text. It can be used to match, search, split, and replace text. I use it here for searching project titles, authors, and code.

Basic Syntax

The basic syntax for regex is:

/searchQuery/flag

Flags control how it searches, the most commonly used and arguably the most useful are "i" and "g", but here is a full list:

  • g: The global flag will check for all matches rather than just one
  • i: The case insensitive flag will match the characters regardless of capitalization
  • m: The multiline flag changes the function of ^ and $
  • u: The unicode flag allows you to use escaped unicode, but will stop other escaped characters
  • y: The sticky flag will only use the last match
  • s: The dotall flag will make the "." character represent all non-whitespace and whitespace characters

At its most simple the searchQuery can be what you want to find (e.g. /how to/i for "how to"). This makes it easy for beginners.

Special Characters

There are special characters that can make your search more complex though. In this way regex can both be very easy to use or very hard to use. A great resource for learning and testing your regex is regexr.

+*?^$\\.[\]{}()|\/ are the special characters with their own special functions. They must have a \ before them to actually search for them (e.g. /func\(\)\{\}/i for func(){}).

Note: If you search for a code snippet, click "code to regex" to automatically add a "\". You can read more about it in plagiarism

Tutorial: Only Show Graphics

Let's define the difference between games and graphics so we can know what to search for. My definition would be that games have user interaction and graphics don't. So to get graphics only we need to exclude:

  • mouseX
  • mouseY
  • pmouseX
  • pmouseY
  • mouseButton
  • mouseIsPressed
  • key
  • keyCode
  • keyIsPressed
  • mouseClicked = function(){}
  • mouseReleased = function(){}
  • mouseDragged = function(){}
  • mouseOut = function(){}
  • mousePressed = function(){}
  • mouseMoved = function(){}
  • mouseOver = function(){}
  • keyPressed = function(){}
  • keyReleased = function(){}
  • keyTyped = function(){}

We can just include each of those variable names in the exclude regex, but I want to take it farther than that to demonstrate the power of regex. One concern is if those variables are properties of an object like: obj.key or obj.   key. That should not be allowed. We can fix that by using a negative look behind

/(?<!\.\s*)(key)/

This gets complex fast doesn't it! Let's break it down. The () are to group characters. So we have one group looking for key. The other group is surrounded by (?<!) which means negative look behind. This means (key) cannot have \.\s* behind it. That code means text that has a . and then zero or more whitespace after the .. The . is represented by ./. The white space is represented by \s, and the * means zero or more of the preceding group (0 or more white space characters).

Have you spotted a problem though? Will thatkey or keythis be counted? The answer is yes. Characters that can be part of a variable name include "_", "A"-"Z", "a"-"z", "$", and "0"-"9". So we will need to prevent those from being in front or behind of key. In other words, we need a character other than those characters in front and behind. Luckily, there is a way to do that.

You can exclude characters with [^ABC] (that excludes A, B, and C), and you can exclude a range with [^A-Z]. So lets plug in every character found in variables: [^A-z0-9$_]. We said we wanted that on both ends, so:

(?<!\.\s*)([^A-z0-9$_]key[^A-z0-9$_])

Sadly, that is not all. Now that we require a character around "key" it won't match if it is at the beginning or even end of the code. Luckily, RegExp supplies a character to represent the position of an end or beginning to a word: \b (word boundary) We are also lucky that all the characters in the variables are word characters. This means we want either no variable characters ([^A-z0-9$_]) or a word boundary (\b) before and after our variable name. How do we do the "or"? Well, it is pretty simple because | represents "or", and () can separate it off from the rest of the code. ([^A-z0-9$_]|\b)

The last problem is if the variable is local. Now that is really difficult to check because it is hard to keep track of the number of {}. and you have to check for both var, and (parameter). It might be possible, but it is not easy. So, let's end here by applying what we have to everything we want to exclude:

  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseX([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseY([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)pmouseX([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)pmouseY([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseButton([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)key([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseIsPressed([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)keyCode([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)keyIsPressed([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseClicked([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseReleased([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseDragged([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseOut([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mousePressed([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseMoved([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)mouseOver([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)keyPressed([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)keyTyped([^A-z0-9$_]|\b))
  • (?<!\.\s*)(([^A-z0-9$_]|\b)keyReleased([^A-z0-9$_]|\b))

As you can see, regex takes some dedication, but it is also very versatile. Luckily, I did this for you, just click the buttons below to automatically fill out the search form.

Icon Guide

Why?

These icons are used in the project cards. They allow the cards to be smaller and more appealing.


Query Rank Icon

The query rank is the rank of the project in the list you are making. Every time a project is added, the query rank increases. This means loading from a top list will create a query rank based on how successful your project was.


Total Rank Icon

The total rank is where the project is placed in KA's API list. If you use top lists, the total rank will be where the project ranks in that whole list. You can use this to look at how far the list was loaded to, and you can use it to see where the project ranks.


Votes Icon

The votes icon precedes the number of votes a project has.


Spin Off Icon

The spin offs icon precedes the number of spin offs a project has.


Flag Icon

The flag icon will precede a number that tells you how many flags a project has. If you hover over the flag icon, you can see the types of flags. This data is most useful for the recent list where flags are more likely to be accurate.

Advanced Search Guide

What is Advanced Search?

If you start a search with the advanced search dropdown shown, more data will be retrieved for the project type, the date created, the number of lines, the number of characters, and the code. You can use this to gain more insight into a project or filter your results better.

Advantages

If you use advanced search many amazing ways to filter and sort are given to you. You can filter and sort by the actual creation date rather than the date it was last updated. You can use regex to include or exclude snippets of code. You can filter the project type to find spin offs, webpages and more. You can also filter and sort by program length that can be measured using characters or lines.

Disadvantages

When you use advanced search, data will have to be retrieved for every individual project which increases the loading time up to 10 fold. This results in a much slower loading time. However, if you add restrictions, such as last updated date, in the normal search, it will increase in speed.

Limitations

Here I will discuss things you can't do with my program and why.

I do not have any searches for the username of the author, program comments, program questions, and program help requests. This is because they would all require a third API request which would take an immense amount of time to load through, especially because these requests have a lot of data in them.

There is also no list for projects that are waiting to be evaluated, or projects that have been evaluated in this program. This is because much of the data needed, such as votes, would only be available with a second API request which would be very slow. If you want to check out a list of these projects, check out YAPEP

Finally, you can't find hidden projects through this program because the API has no information available for hidden projects.

User Made Lists Guide

What are user made lists?

User made lists are lists that can be created by anyone that uses this program. You can use these lists so you don't have to load a search that others have already done. The loading of a user created list is much faster than a normal search because the only thing limiting it is the KA Infinite Loop Protect. You can also sort these lists by all of the data they have on their card which allows for endless ways to find projects.

How can you use them?

It is very easy to use lists others have created. First go to the page "User Made Lists" through the sidebar. Then you will have to wait a short while for the lists to load in. Once they are loaded, you can type keywords into the filter bar to filter the lists to what you want. When you find one you like, click "show list", and the list will appear. You can sort the list using the sorting form at any time.

How can you make your own?

Making your own is slightly more difficult, but by doing so you are saving the list for yourself, and for others. Not to mention adding the sorting functionality.

The first step is to load the list you want through the project search page. Once you have stopped the loading and have edited your list the way you want, click "copy" in the search menu.

Fill out the pop-up that appears with the name and a description. Make sure that the description says what your list includes, but nothing else so the keyword filter works properly. A good example would be: "This list shows graphics and animations from 2020". A bad example would be: "This list shows projects from 2020 that aren't games or interactive". Finally, click "finish".

The data will now be copied to your clipboard so you can now paste it. You should paste it into a new project and save the project. Make sure that the data is the only code, if there is anything else, the list won't show up in KA Search when you finish the last steps.

Next, copy the numbers from the end of the URL (this program's number is 5263884573032448). I need this number so I can use the API to get the code of the program. Paste that number (and only that number) into a new comment under the "user made lists" thread in the T&T.

If you did everything right, the next time you open up the user made lists, your list should be there.

Search Guide

How do you make a basic search?

Open the sidebar and click "Project Search". Then fill out the form with what you want, such as date last updated, votes, and spin offs. Next click the magnifying glass in the bottom right corner to start the search. The search page will be brought up along with a drop down menu where you can pause the search, end the search, and copy the results. The search will continue even if you switch tabs or windows until you pause it.

Note: the search fields begin with values that will show every project. Any changes to the values will be saved until the program is reloaded.

Searching for author, title, and code snippets

With this program you can search for the author's nickname (or kaid), the title, and code snippets. To do this, click the + button under where you want it added. Loading Issues

Has your loading stopped? Then you came to the right place. There are two possible reasons it stopped.

The most common reason is that the API stalled for a little bit. I don't know why it happens, but the API data just takes a long time to fetch. Luckily, if you wait a while, this will go away like it never happened.

Where to search fieldset

The where to search fieldset is a very important one. Here you can choose what API list you want to load from. The lists are the same ones available to you in the browse section of KA computer programming (excluding the hot list). You should use the top lists when you want to find the top of something because the better programs are nearer to the start of the list. You can also use the recent list to browse projects being made right now. You can adjust the amount of lines in the program and you can filter out spin offs so this list will return projects that you are interested in. This is a great alternative to the hotlist when the hotlist has a lot of projects that swamp out any new ones trying to get on it.

You can also select where you want to start in that list. This is a great thing for when you are searching for one specific project and need to turn off your computer. Just write down the project number you were on and put it in this box to start again from that point.

Restoring Old Searches

Whenever you end a search, whether it is by clicking "end search" or by switching to a different page, the projects will be deleted to relieve your computer of some strain. So how can you return to that list?

If you go to the project search page, there will be a button at the top that you can click to restore your previous search query and list.

How does this work?

Where does the information come from?

I retrieve project details, user made lists, and browse lists from Khan Academy's API. API stands for application programming interface and it allows people to access data.

How can you retrieve information from the API?

The function that allows you to retrieve stuff from the API uses JQuery, so the first step is to add it.

The function you use is: $.getJSON("link", func)

The function (func) is the easy part. This function will be called when you get the data. It is given one argument that contains all the data you need for example:

var useTopListData = function(data){};
$.getJSON("link", useTopListData);

The links come with experience and exploration, but they all must be followed by "&callback=?". For example, here is the link for the top list: "https://www.khanacademy.org/api/internal/scratchpads/top?topic_id=xffde7c31&limit=10&sort=5&callback=?" The limit is set to 10 in this link, and it determines how many projects will be included in the API request. The sort is set to 5 and represents what browse list we are retrieving from (5 is the top list).

Now that we have the link, let's put it all together:

var useTopListData = function(data){};
$.getJSON("https://www.khanacademy.org/api/internal/scratchpads/top?topic_id=xffde7c31&limit=10&sort=5&callback=?", useTopListData);

Note: the getJSON method is asynchronous meaning that the code it resides in does not wait for when it retrieves data.

How can you get more than one page?

The problem now is that with that we can only retrieve the top 10 projects, how can we retrieve more? That is what a cursor is for. Each list request will return data.cursor that we can use in the next request like:

$.getJSON("https://www.khanacademy.org/api/internal/scratchpads/top?topic_id=xffde7c31&limit=10&sort=5&cursor="+prevData.cursor+"&callback=?", useTopListData);

Guide to Plagiarism

How can I use this to check for plagiarism?

You can use this program to check for plagiarism in two ways:

You can use code snippets from commonly plagiarized projects to catch plagiarism. This is especially useful in the recent based on official project list where many tutorial projects are plagiarism.

You can also use this to find an original project if you suspect a project is plagiarism. To do this, you can use titles, authors, and code snippets from the project that you suspect is plagiarism. This search is best done in one of the top lists.

How to use code snippets?

For this I will use the commonly plagiarized project by Kaloyan as an example.

The first question is, what portions of code should I use as my code snippets? Well, the code snippets must be small and unique.

The snippets can't be large because that makes it easy for one change (such as a color change) to mess up the whole snippet. If we keep them small, then those chances are reduced, typically one line is a good size. A good example of when a size is too large is in the Kaloyan project:
drawFish(200, 100, 118, 74, 70, 30, color(162, 0, 255), color(162, 0, 255), color(162, 0, 255));
drawFish(220, 180, 150, 65, 70, 5, color(162, 0, 255), color(162, 0, 255), color(250, 85, 255));
drawFish(285, 265, 185, 74, 70, 20, color(162, 0, 255), color(162, 0, 255), color(50, 145, 95));
What are the chances one of those are modified? I'd say it is a very high chance.

The code snippets must also be unique to that project so projects we aren't looking for don't show up. If we used var mouseClicked = function(), there would be a lot of matches that aren't plagiarism.

Now that you have a line, copy it. You will want to make a new "include" search for the "code" fieldset. Paste in the code, and click the "code to regex" button. This will escape (add a "\") before all of the special regex characters so they don't function and mess up our search. If you get an error, it is likely because you didn't press this button.

The only question that remains is how many lines should you add. You should really add as many as you want so that you have a higher chance of catching plagiarism. Surprisingly, regex search is fast enough to handle more than ten complex searches at once, so I would not worry about your simple searches. You can learn more about regex here

The matches returned may not be plagiarism. Make sure to check with your own eyes before you flag anything.

Advantages

This program can dig deep into the depths of the top list, much further than man would ever be able to go. This gives it the ability to check vast quantities of projects very thoroughly.

Disadvantages

Searching for projects with code snippets takes a very long time because it requires the second data search (see more here). However, there are ways to help improve speed.

When you are trying to find an original, it is very hard to increase the speed since you are only looking for one project in the whole top list. However, you can still decrease the speed a little bit. If you are lucky, the project you think that is plagiarism might have hints at the author's name or the title. If that is the case, use it because those will stop a lot of second searches. The last area you can change to improve speed is the last updated date. Typically the original won't be updated after the copy was made, so that can save us a bit of time too.

Finding copies of a program can be much faster. The concentration of plagiarism increases the speed, so the best use of it would be for the recent based on official projects list (tutorial projects). If you use this list, you can also use title search to speed up the search by filtering out projects that aren't of the type you want (e.g. Fish Tank)
